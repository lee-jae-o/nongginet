### **📌 개요 및 사용 기술**

---

본 프로젝트는 농림축산식품부 공공데이터를 활용하여 농기계 임대, 구매, 안전 관리를 통합적으로 지원하는 플랫폼 ‘농기넷’을 개발한 것입니다. 

FastAPI와 React를 기반으로 LangChain을 활용한 RAG 챗봇 기능을 포함하였습니다.

- **Backend**: FastAPI (Python)
- **Frontend**: React, Chakra UI
- **AI/ML**: LangChain, OpenAI API(GPT-4o Mini), ChromaDB
- **Database**: MySQL, MongoDB
- **Server**: AWS EC2
- **Container**: Docker
- **Editor**: Cursor


<div>&nbsp;</div>


### 🤔 주제 선정

---

2025년 농림축산식품 공공데이터 활용 창업경진대회를 준비하면서, 농기계 관련 정보가 임대, 구입, 안전지침 등 항목별로 제각각 흩어져 있다는 점을 확인했습니다.

정보 제공 기관도 제각각이고, 접근 방식이나 형식도 달라 사용자 입장에서 한눈에 정보를 파악하고 활용하기 어려운 구조였습니다.

이런 불편함을 해소할 수 있는 서비스가 필요하다고 생각했고, ‘농기넷’ 이라는 서비스를 만들게 되었습니다.

### 📐 개발 환경 및 시스템 구성

---

![<농기넷 시스템 구성도>](DATA2025_(2).jpg)

<농기넷 시스템 구성도>

백엔드는 FastAPI로 구현하여 회원 관리, 게시판, 챗봇, 농기계 정보 제공 등 핵심 기능을 담당하며,

프론트엔드는 React와 Chakra UI를 사용해 화면을 구성하였습니다.

데이터는 정형 데이터(MySQL)와 비정형 데이터(MongoDB Atlas)로 구분하여 저장하고 있으며,

공공데이터 API에서 수집한 정보를 JSON 형태로 가공해 활용하고 있습니다.

또한, 농림축산식품부의 ‘2025년 농기계임대 사업시행지침’ PDF 문서를 벡터화하여 저장하고,

LangChain 기반 RAG구조를 활용해 해당 문서 내용을 기반으로 질문에 응답할 수 있도록 구성했습니다.

전체 서비스는 AWS EC2 환경에서 Docker와 Nginx를 이용해 통합 배포하였고,

SSL 인증도 완료하여 HTTPS 접속이 가능하도록 구성했습니다.

🗺️ **농기계 임대소 지도**

---

![스크린샷 2025-08-10 231725.png](%EC%8A%A4%ED%81%AC%EB%A6%B0%EC%83%B7_2025-08-10_231725.png)

Kakao 지도 API를 활용하여, 공공데이터포털의 ‘전국농기계임대정보표준데이터’를 기반으로 전국 임대사업소의 위치를 시각화하였습니다.

마커 클릭 시 사업소명과 ‘상세보기’ 버튼이 표시되며, 상세 모달에서는 보유 기종, 전화번호, 주소 등의 정보를 확인할 수 있습니다.

즐겨찾기, 기계 종류 필터링, 검색 기능도 함께 제공합니다.

🛒 **농기계 구매 정보**

---

![스크린샷 2025-08-10 231835.png](%EC%8A%A4%ED%81%AC%EB%A6%B0%EC%83%B7_2025-08-10_231835.png)

농사로의 ‘농기계 구입정보’를 활용하여, 기종명, 연도, 가격 범위를 기준으로 농기계를 필터링할 수 있습니다.

필터 결과는 카드 형태로 제공되며, 기종별 평균 가격은 막대 차트로 시각화됩니다.

각 항목은 즐겨찾기에 추가할 수 있습니다.

🦺 **농기계 안전 정보**

---

![스크린샷 2025-08-10 231927.png](%EC%8A%A4%ED%81%AC%EB%A6%B0%EC%83%B7_2025-08-10_231927.png)

안전이용 지침, 사고사례, 퀴즈의 3가지 탭으로 구성되어 있습니다.

안전이용 지침은 제목, 내용, 사고유형, 기종명 등의 항목을 표 형식으로 나열하며, 사고사례는 제목 클릭 시 상세 내용을 확인할 수 있습니다. 두 항목 모두 제목, 내용, 기종명을 기준으로 검색 기능을 제공합니다.

퀴즈 탭에서는 안전 관련 내용을 기반으로 한 객관식 문제를 직접 풀어볼 수 있습니다.

### 📈 향후 발전 방향

---

- 현재 단일 문서 기반 구조를 멀티 문서·멀티 도메인 검색으로 확장

- 대화 내용 기반의 후속 질문 처리 및 맥락 유지 기능 강화

### 💭 회고 및 소감

---

서류 붙고 발표까지 간 건 처음이라 준비 단계부터 계속 긴장했음. 지금 생각해보면 조금 더 편하게 했어도 되지 않았을까 싶음. 원래는 그냥 대학생 공모전 정도로 생각했는데, 막상 가보니 직장인들이 절반 이상이고, 몇몇 팀 발표 내용을 엿들어보니 내가 예상한 ‘대학생 공모전’ 수준이 아니라 진짜 ‘창업 설명회’에 가까웠음. 그래서 이때부터 ‘아, 상 받기는 힘들겠다’ 싶었음.

예상대로 상은 못 받았지만, 준비 과정에서 이것저것 찾아보고 수정해 나가는 시간은 꽤 재미있었음. 내가 만든 걸 처음 보는 사람들 앞에서 직접 설명해본 경험도 나름 뿌듯했음.

다시 한다면 단순히 “플랫폼 하나 만들었어요” 수준이 아니라, 마케팅부터 실제 서비스 운영까지 다 해보면 훨씬 재미있을 거 같다는 생각이 듦.
